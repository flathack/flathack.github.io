FL-Lingo

FL-Lingo ist ein Relokalisierungs-Tool fuer Freelancer-Mods. Es vergleicht eine aktuelle modifizierte Installation mit einer sauberen Referenzinstallation, stellt bekannte Vanilla-Texte automatisch wieder her, exportiert mod-spezifische Inhalte fuer externe Uebersetzungen und schreibt die lokalisierten Ergebnisse mit Backup-Strategie zurueck in die Resource-DLLs.

Relokalisierung DLL-Workflow In Entwicklung

Schnellzugriff

  • GitHub: github.com/flathack/FL-Lingo
  • Projektdatei: .FLLingo
  • UI-Sprachen: EN, DE, FR, ES, RU

Uebersicht

FL-Lingo ist fuer den typischen Fall gebaut, dass eine Freelancer-Mod eine lokalisierte Installation wieder auf Englisch oder andere mod-eigene Texte zuruecksetzt. Das Programm vergleicht eine modifizierte Spielinstallation mit einer sauberen Referenzinstallation in der Zielsprache und klassifiziert Eintraege in automatisch wiederherstellbar, bereits lokalisiert, manuell uebersetzt oder mod-spezifisch.

Kurzinfos

  • Typ: Desktop-Anwendung fuer Windows und Linux
  • Rolle: Compare-, Export-, Import- und Apply-Workflow
  • Status: aktiv in Entwicklung

Installation

  • Python 3.11+ verwenden
  • Virtuelle Umgebung anlegen und `python -m pip install -e .[all]` ausfuehren
  • Start ueber `python launch.py` aus dem Projektordner

Schnellstart

  • Aktuelle modifizierte Freelancer-Installation waehlen
  • Saubere Referenzinstallation in der Zielsprache laden
  • Vergleich starten, mod-spezifische Eintraege exportieren, Uebersetzungen importieren und anwenden

Wichtige Funktionen

  • Simple Mode fuer gefuehrte drei-Spalten-Arbeitsschritte
  • Expert Mode mit Filtern, DLL-Analyse, manuellen Korrekturen und Terminologie
  • Automatische Backups vor Schreibvorgaengen und Restore aus der App

Technische Daten

  • Arbeitet mit String Tables, Infocards und DLL-Referenzen aus `EXE/freelancer.ini`
  • Unterstuetzt Terminologie-Dateien pro Zielsprache und Projektdateien im Format `.FLLingo`
  • Tests laufen ueber `pytest`, GUI-Smoke-Tests headless mit `QT_QPA_PLATFORM=offscreen`

Sicherheit

  • Vor dem Anwenden werden automatische Backups erzeugt
  • DLLs werden nicht blind ueberschrieben, sondern nach sicherster Schreibstrategie behandelt
  • Empfohlen ist das Testen auf einer Kopie des Spiels mit sauberer Referenzinstallation

Projektstruktur

  • `launch.py` startet die App mit zentralen Defaults
  • `src/flatlas_translator/` enthaelt UI, Katalog-, DLL- und Schreiblogik
  • `Languages/`, `data/help/`, `data/` und `tests/` decken UI-Texte, Hilfe, Terminologie und Tests ab
